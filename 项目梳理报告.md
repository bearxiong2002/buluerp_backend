# BlueERp 项目梳理报告

> 生成时间：2026-06-16

## 一、项目概述

BlueERp（布鲁克ERP）是一套**塑料积木/玩具制造行业的 ERP 管理系统**，基于 RuoYi-Vue v3.8.9 框架二次开发，覆盖从订单接单、产品设计、模具管理、生产排产、采购、库存到包装发货的完整制造业务流程。

## 四、核心业务模块

### 4.1 订单管理（erp_orders）

订单是整个系统的驱动核心，采用 **16 阶段状态流转**：

```
审核不通过 ← 创建(未审核) → 待设计 → 设计中 → 待定制布产计划
→ 布产计划已定制(待排产) → 排产中 → 生产完成(待采购完成)
→ 已齐料入库(待套料) → 套料中 → 套料完成(待拉线)
→ 拉线组包中 → 拉线完成(待包装) → 包装中 → 已发货 → 已完成
```

状态流转由不同部门角色控制：设计部、仓库、拉线部、销售部、管理员。

### 4.2 产品设计（erp_design_patterns / erp_design_style / erp_design_sub_pattern）

设计模块连接产品与订单，管理积木造型的设计细节：
- **设计总表**（DesignPatterns）：将产品与订单关联
- **设计造型**（DesignStyle）：模具编号、LDD 编号、颜色、数量、图片
- **设计子模式**（DesignSubPattern）：设计总表下的子项明细

另有 BOM 模块（erp_bom / erp_bom_node / erp_bom_knowledge）支持 AI 辅助 BOM 生成，含向量嵌入。

### 4.3 模具管理（erp_mould / erp_manufacturer / erp_mould_house）

模具是制造核心资产，跟踪 **6 阶段生命周期**：
1. 新模(排产中) → 2. 新模完成(待试模) → 3. 验收合格(已入库)
4. 模具故障送修中 → 5. 维修好返厂待试模 → 6. 已外发

关联实体：模具厂商（manufacturer）、模房（mould_house，存储位置）。

### 4.4 物料管理（erp_material_info / erp_material_type）

物料资料包含图纸引用、模具编号、模穴数、材料类型、3D 模型 URL。物料类型管理颜色和重量参数。

### 4.5 生产管理（erp_production_schedule / erp_production_arrange）

- **布产**（ProductionSchedule）：制定生产计划，关联订单、产品、模具、颜色、模穴数、产量、周期时间、供应商，含审核流程
- **排产**（ProductionArrange）：安排具体生产，管理模具产出、计划产量、排期、完工时间

### 4.6 采购管理（erp_purchase_info / erp_purchase_collection / erp_purchase_order）

- **外购资料**（PurchaseInfo）：外购件目录（编码、单价、供应商）
- **采购计划**（PurchaseCollection）：按订单制定采购计划，含审核流程
- **采购订单**（PurchaseOrder）：实际采购执行 + 发票管理

### 4.7 库存管理（三组"当前库存 + 变动日志"表）

| 类型 | 库存表 | 变动日志表 |
|------|--------|-----------|
| 产品库存 | erp_product_inventory | erp_product_inventory_change |
| 零件库存 | erp_part_inventory | erp_part_inventory_change |
| 包装材料库存 | erp_packaging_material_inventory | erp_packaging_material_inventory_change |

### 4.8 包装管理（erp_packaging_list / erp_packaging_bag / erp_packaging_detail）

三层级包装结构：
- **分包表**（PackagingList）：订单级别的包装方案（含审核）
- **分包袋**（PackagingBag）：每袋的重量、规格、数量
- **分包明细**（PackagingDetail）：袋内零件明细（模具号、物料类型、颜色、用量）

## 五、审核系统

四种审核类型，各自独立开关控制：
1. **订单审核**（code=1，角色：order_auditor）
2. **采购审核**（code=2，角色：purchase_auditor）
3. **布产审核**（code=3，角色：production_auditor）
4. **分包审核**（code=4，角色：packaging_auditor）

审核状态：0=待审、1=审核中、-1=不通过、2=通过。审核记录保存在 erp_audit_record。

## 六、通知与实时推送

- 29 种通知类型，覆盖设计、PMC、采购、注塑、仓库、拉线、销售各部门的审核工作流
- WebSocket（/websocket/{token}）实时推送，支持 JWT 认证、按用户投递、批量消息
- 通知切面 @MarkNotificationsAsRead 自动标记已读

## 七、用户与角色体系

**部门角色**：sell_dept（销售）、design_dept（设计）、injectionmolding_dept（注塑）、pmc_dept（PMC）、warehouse（仓库）、wirestaying_dept（拉线）、purchase_dept（采购）

**审核角色**：order_auditor、purchase_auditor、production_auditor、packaging_auditor

**预设账号**：8 名员工（振凯-销售、家焕-厂长、一帆-注塑、凯涛-工程、杰旭-仓库、旺达-拉线、锐德-PMC、巧佳-采购）+ 3 个 BOSS 超级管理员 + 3 个备用账号

## 八、横切关注点

- **操作日志**：OperationLogAspect 自动拦截所有 ERP Controller 的 @ApiOperation 方法，@AutoLogIdentifier 标记追踪字段，@AutoLogIgnore 排除字段
- **数据校验**：自定义校验组（Save/Update）+ @NullOrNotBlank 校验器
- **3D 模型**：支持 STP 转 GLTF 的 Python 脚本（CadQuery Conda 环境），独立于代码仓库
